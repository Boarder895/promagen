## 4. Open decisions — needed before more work

### Decision 1 — Stripe deletion

**Files queued (6):**
```
app/api/stripe/checkout/route.ts
app/api/stripe/portal/route.ts
app/api/stripe/debug/route.ts
app/api/stripe/webhook/route.ts        ← DANGEROUS
lib/stripe/clerk-session.ts
lib/stripe/stripe.ts
```

**The risk:** the webhook receives `customer.subscription.updated`, `invoice.payment_failed`, `customer.subscription.deleted` events from Stripe. Deleting it means subscription state changes silently fail to propagate. If any active Stripe subscriptions exist, customers continue to be billed but Promagen has no record of state changes.

**The yes/no needed:**

> "I have zero paying Stripe customers AND no active Stripe products/prices I intend to keep selling."

If yes → delete in one pass.
If no → leave Stripe in place permanently or until status changes.
**Check Stripe dashboard before answering.**
